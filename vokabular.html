Mein Git Profi-Vokabular

Grundbegriffe

Branch (Ast/Zweig)
Eine isolierte Kopie deines Codes, um sicher an neuen Features zu arbeiten, ohne die Hauptversion zu stören.
Main
Der Standard-Zweig deines Projekts (die "Hauptstraße").
HEAD
Der Zeiger, der angibt, wo du dich gerade in deinem Git-Verlauf befindest ("Du bist hier"-Marker).

Aktionen & Befehle

Stage (git add)
Änderungen für den nächsten Commit vormerken (die "Versandbox" füllen).
Commit
Einen festen Speicherpunkt in der Historie erstellen (das "Paket" versiegeln).
Push
Lokale Commits auf einen Remote-Server wie GitHub hochladen.
Origin
Der Spitzname für die Adresse deines Online-Repositories (z.B. auf GitHub).

origin ist einfach nur ein Spitzname für die Internetadresse (URL) deines Repositories auf GitHub.

Anstatt jedes Mal die lange, komplizierte Adresse tippen zu müssen (wie https://github.com/DeinName/DeinProjekt.git), hat Git beim ersten Einrichten diesen langen Link unter dem kurzen Namen origin gespeichert.

Lokal
Dein Ordner auf dem Computer.
Remote (origin)
Die „Gegenstelle" im Internet (dein GitHub-Konto).

Profi-Begriffe (Angeber-Vokabular)

Refactor
Code aufräumen oder umstrukturieren, ohne seine Funktion nach außen hin zu ändern.
Beispiel: "Refactored code: removed redundant sections"
Redundant
Überflüssig oder doppelt vorhanden.
