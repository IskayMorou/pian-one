What are tokens in this context?

A named value that replaces a raw value

A token is a named value that replaces a raw value. Instead of #FF3F66 scattered across 40 CSS rules, you have one name — --color-accent — and everything references that name. The token is the name.

--color-bg, --color-text, --color-accent — those are your tokens.

Heading/lg/Bold describes a combination of properties — size, weight, line-height. That's useful, but it sits one level above the tokens.

The cleaner mental model has two layers:

Layer 1 — Tokens (the raw values)
                   
        font-family: Recursive
        font-size-lg: 52px
        font-weight-bold: 700
Layer 2 — Text styles (combinations of tokens)
        Heading/lg/Bold → Recursive, 52px, 700

A token is a named value. The name is the token. The raw value is what the token holds. So in Layer 1: font-size-lg: 52px 52px is the raw value. font-size-lg is the token. You never write 52px directly anywhere — you always  write font-size-lg. That indirection is the whole point.

Is this the same as Tailwind's utility classes?

modifier classes is also tailwind style? tailwind way to organize?

Yes, exactly — that's the core idea behind Tailwind. Small, single-purpose classes that you compose in HTML rather than writing custom CSS for every combination.

Tailwind just takes it much further — it generates hundreds of utility classes for everything (spacing, color, borders, shadows...) so you rarely write CSS at all.

What you're building is the same concept but hand-crafted and minimal — only the tokens and utilities your system actually needs. That's actually a great way to understand why Tailwind works before using it as a black box.

The class is now intention-readable (body-lg tells you what it is) but value-opaque (you can't see 20px/1.35 without jumping to :root). The raw version was the opposite — values visible, meaning implicit.

This is why design systems always need two things together:

  • The token file (:root) as the source of truth
  • Documentation or Figma as the readable reference

When you're deep in CSS and want to know "what size is body-lg?" — you look at :root, or eventually at the Figma file where Body/lg = 20px is right there visually.

The abstraction cost is real but it's a one-time learning curve. After a week you stop thinking body-lg = 20px and start thinking body-lg = comfortable reading size. That's actually the goal — thinking in scale, not pixels.
