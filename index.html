Basics

Repository

Auf deinem Computer hast du normale Ordner für Texte oder Bilder. Ein Repository auf GitHub ist fast genauso, aber es hat ein „Gedächtnis“. Es speichert nicht nur den aktuellen Stand deiner Dateien, sondern merkt sich jede einzelne Änderung, die du jemals gemacht hast.

Die „Zeitmaschine“ (Version Control)

Das ist das Beste an einem Repository: Wenn du heute Code schreibst und morgen merkst, dass alles kaputt ist, kannst du das Repository einfach fragen: „Zeig mir mal, wie der Code gestern aussah!“ Du kannst per Knopfdruck in der Zeit zurückreisen und Fehler rückgängig machen.

Die Zusammenarbeit

Stell dir vor, du arbeitest mit drei Personen an einem Projekt

Ohne Repository müsstet ihr euch ständig Dateien per E-Mail oder USB-Stick schicken. Chaos!

Mit einem Repository hat jeder eine Kopie des Ordners auf seinem Computer. Wenn jemand fertig ist, „lädt“ er seine Änderungen ins Repository auf GitHub hoch (das nennt man Push). Die anderen können sich die neuen Teile dann einfach „ziehen“ (Pull).

Die wichtigsten Begriffe kurz erklärt:

  • Commit

    Das ist wie ein „Speicherpunkt“ in einem Videospiel. Du sagst dem Repository: „Diesen Stand bitte merken!“

  • README

    Das ist meistens eine Textdatei im Repository, die erklärt, worum es in dem Projekt überhaupt geht (quasi der Klappentext eines Buches)

  • Public vs. Private:

    Ein Repository kann öffentlich sein (jeder auf der Welt kann deinen Code sehen und davon lernen) oder privat (nur du und deine Freunde haben Zugriff).

Cosole

  • mkdir MeinNeuerOrdner
  • ls

    Tippe ls ein, um dir alle Dateien und Ordner an deinem aktuellen Ort anzeigen zu lassen.

  • cd

    Stell dir das Terminal wie einen Wanderer vor: Mit mkdir hast du ein neues Haus gebaut, und mit cd gehst du jetzt durch die Tür hinein.

    Tippe einfach cd gefolgt von dem Namen deines Ordners ein

    cd MeinNeuerOrdner & enter

  • Einen Schritt zurück:

    cd ..

    (Die zwei Punkte bedeuten: Geh eine Ebene höher, also wieder aus dem Ordner raus.)
